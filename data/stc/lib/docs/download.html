Installation and Startup

For the Apple(aqua) version, drag the stc-v5.3.0.app folder into /Applications and double click on stc-v5.3.0 to start it.

For the X11 versions, you need to enter these commands:

  1. Untar the software if your browser has not already done so. 
    > tar xvf stc-v5.3.0-build.tar
  2. Run the installation script (decide where to put the software eg, /usr/local). 
    > cd stc-v5.3.0-build
> ./Install
  3. Update your home .cshrc file (or equivalent startup file). 
    > alias stc "wish /usr/local/stc-v5.3.0/bin/stc"
  4. Start a new window and type "stc". If it works, remember to remove the stc-v5.3.0-build and stc-v5.3.0-build.tar files.

Additional Notes and Troubleshooting

  1. The STC window did not appear when i tried to start it.

    Perhaps your environment is not set up to find the tcl interpretter called 'wish'. Do you get the message "Command not found" when you type 'wish'? Mac users can install tcl/tk from http://tcltkaqua.sourceforge.net/, sgi users could go to http://freeware.sgi.com.

  2. You can also move stc to wherever you want (eg, /opt) but you then have to set the STC_HOME environment variable. 
    > setenv STC_HOME /opt/stc-v5.3.0
> alias stc "wish /opt/stc-v5.3.0/bin/stc"
  3. You should be able to run stc remotely but make sure you pass the "-X" flag to ssh which allows X11 forwarding.
